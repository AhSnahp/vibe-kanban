# AI Memory

Architecture documentation auto-loaded into Claude Code sessions via the ai-memory plugin.

Files are generated by `/ai-memory-retrofit` and can be regenerated anytime.
Update manually for nuances the scanner can't detect, or re-run retrofit after major changes.

## Files

| File | Purpose |
|------|---------|
| `architecture.md` | System overview, tech stack, crate/package map, deployment modes |
| `data-flow.md` | Database schema (ER diagram), API routes, request lifecycle |
| `component-map.md` | Frontend component hierarchy, stores, hooks, design system |
